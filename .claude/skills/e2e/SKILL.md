# E2E Testing

Run end-to-end tests for all Slack → Graphein integration paths.

Arguments: $ARGUMENTS

The arguments should contain the Slack workspace URL and the Graphein app URL.
If not provided, ask the user for both URLs.

## Prerequisites

- The Graphein dev server must be running
- Slack must be accessible in a browser (user logged in)
- Playwright MCP must be available
- The dev database must be accessible via `bun run tools/run-sql.ts`

## Overview

There are 5 integration paths to test, all exercising the `resolveSlackUserToDb` helper:

1. **Add Task shortcut** — message shortcut → modal → task created
2. **Add Snippet shortcut** — message shortcut → snippet saved
3. **Add Kudos shortcut** — message shortcut → kudos saved (must be in a kudos channel)
4. **Snippet event listener** — post message in snippet channel → auto-saved
5. **Kudos event listener** — post message in kudos channel → auto-saved

## Channel Configuration

Query configured channels from the database:

```sql
SELECT sc.slack_channel_id FROM snippet_channels sc;
SELECT kc.slack_channel_id FROM kudos_channels kc;
```

Use these channel IDs for testing event listeners and the kudos shortcut.

## Browser Automation Notes

These patterns were validated through manual testing. Follow them exactly.

### General

- Use `mcp__plugin_playwright_playwright__browser_snapshot` (no depth limit) to inspect page state
- Use `mcp__plugin_playwright_playwright__browser_navigate` to open URLs
- Slack often shows "Information" dialogs that block interaction — always check for and dismiss them by clicking the "Close" button

### Slack Message Actions (Shortcuts)

To trigger a message shortcut:

1. Navigate to the target Slack channel
2. Hover over the target message to reveal the action toolbar
3. Click the "More actions" button (⋯) in the toolbar
4. In the menu, find and click the shortcut name (e.g., "Add Task", "Add Snippet", "Add Kudos")
5. If the shortcut is under a submenu like "Connect to apps", click that first, then click the shortcut

### Sending Messages in Slack

To post a message with a mention:

1. Navigate to the target channel
2. Click the message input area
3. Type the message content using `browser_type` with `pressSequentially` mode
4. For @mentions: type `@` followed by the username, wait for the autocomplete picker, then click the correct suggestion
5. **Important**: After selecting a mention from the picker, use `browser_type` (pressSequentially) to append remaining text — do NOT use `browser_fill` as it replaces the entire input including the mention
6. To send: click the **"Send now"** button directly — do NOT press Enter (it adds a newline when the formatting toolbar is active)

### Graphein Web UI

- Navigate to the Graphein URL to verify entries
- The UI uses htmx with `hx-boost` for SPA-like navigation
- Use the period/date filters and entity filters to find specific entries
- Verify entries show correct poster name, timestamp, and content

## Test Execution

Run all 5 tests sequentially. For each test, record a PASS/FAIL result.

### Test 1: Add Task Shortcut

1. Navigate to any Slack channel (prefer a general/test channel, NOT snippet/kudos channels)
2. Find a recent message (or post a new test message)
3. Trigger the "Add Task" shortcut on that message
4. The modal should open with:
   - A pre-filled title (generated by Gemini)
   - The original message as a quote
   - User/group assignment selectors
5. Submit the modal
6. **Verify in Graphein**: Navigate to the Tasks page, confirm the new task appears with the correct title
7. **Verify in DB**:
   ```sql
   SELECT id, title, slack_permalink, created_at FROM tasks ORDER BY created_at DESC LIMIT 1;
   ```

### Test 2: Add Snippet Shortcut

1. Navigate to any Slack channel
2. Find a recent message with a mention (or post one like "@someone test snippet")
3. Trigger the "Add Snippet" shortcut on that message
4. The snippet should be saved immediately (no modal)
5. **Verify in Graphein**: Navigate to Snippets page, confirm the entry appears
6. **Verify in DB**:
   ```sql
   SELECT id, content, slack_permalink, created_at FROM snippets ORDER BY created_at DESC LIMIT 1;
   ```

### Test 3: Add Kudos Shortcut

1. Navigate to a **kudos-configured channel** (from `kudos_channels` table)
2. Find a recent kudos message (or post one like "@someone thanks for helping!")
3. Trigger the "Add Kudos" shortcut on that message
4. It should either save the kudos or show "already added" if the message was already processed by the event listener
5. **Verify**: If newly added, check in Graphein Kudos page and DB:
   ```sql
   SELECT id, slack_permalink, created_at FROM kudos ORDER BY created_at DESC LIMIT 1;
   ```
6. **Note**: If triggered on a non-kudos channel, it should show an error message — this is correct behavior

### Test 4: Snippet Event Listener

1. Navigate to a **snippet-configured channel** (from `snippet_channels` table)
2. Post a new message with a mention, e.g., "@someone This is a test snippet via event listener"
3. Wait ~5 seconds for the event to be processed
4. **Verify in DB**:
   ```sql
   SELECT id, content, posted_by_id, created_at FROM snippets ORDER BY created_at DESC LIMIT 1;
   ```
5. **Verify in Graphein**: Navigate to Snippets page and confirm the entry appears

### Test 5: Kudos Event Listener

1. Navigate to a **kudos-configured channel** (from `kudos_channels` table)
2. Post a new message with a mention, e.g., "@someone thanks for the great work!"
3. Wait ~5 seconds for the event to be processed
4. **Verify in DB**:
   ```sql
   SELECT k.id, k.posted_by_id, k.created_at, ke.message
   FROM kudos k
   JOIN kudos_entries ke ON ke.kudos_id = k.id
   ORDER BY k.created_at DESC LIMIT 1;
   ```
5. **Verify mentioned users**:
   ```sql
   SELECT kemu.user_id
   FROM kudos_entry_mentioned_users kemu
   JOIN kudos_entries ke ON ke.id = kemu.kudos_entry_id
   JOIN kudos k ON k.id = ke.kudos_id
   ORDER BY k.created_at DESC LIMIT 5;
   ```
6. **Verify in Graphein**: Navigate to Kudos page and confirm the entry appears with correct mentions

## Important Rules

- Always verify both in the **Graphein UI** and the **database** for each test
- If a Slack "Information" dialog appears, dismiss it before continuing
- Use `pressSequentially` (not `fill`) when typing in Slack message input after a mention
- Click "Send now" button to send messages (not Enter key)
- Wait a few seconds after posting messages for event listeners to process
- Take snapshots liberally to verify page state before and after actions
