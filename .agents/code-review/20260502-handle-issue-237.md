# 20260502-handle-issue-237

## Context

Issue: https://github.com/nownabe/graphein/issues/237

### Background

The MCP OAuth authorization flow does not resume after Slack login when the user starts `/oauth/authorize` without an active Graphein session. After `/auth/slack/callback`, the user is always redirected to `/tasks`, so the OAuth authorization request is lost.

### Summary

Persist a validated same-origin `return_to` URL as a cookie during `/auth/slack` and redirect to it after successful `/auth/slack/callback`. Stale cookies from abandoned logins are cleared. Tests cover the full flow.
