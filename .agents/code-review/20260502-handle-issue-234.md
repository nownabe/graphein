# 20260502-handle-issue-234

## Context

Issue: https://github.com/nownabe/graphein/issues/234

### Background

`GrapheinOAuthProvider.exchangeRefreshToken()` passes `resource?.toString() ?? ""` into `oauthService.consumeRefreshToken()`. The service then requires exact equality against the stored refresh token resource. Because OAuth refresh requests may omit the optional `resource` parameter, a valid refresh token fails when the client does not send `resource`, even though the stored token already contains the bound MCP resource.

### Summary

Make the `resource` parameter optional in `consumeRefreshToken()`. When omitted, skip the resource equality check and reuse the stored resource. Update the auth provider to pass `undefined` instead of `""`. Add integration and E2E tests for refresh without resource.
